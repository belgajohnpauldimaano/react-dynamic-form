VisualBasement React Exam - Dynamic Form

Goal: Create a form dynamically based on data fed by a REST API.

Features

1. Fetch form fields

Retrieve data from API and render the form on page load. There are 5 different types of data. You should be able to render the appropriate fields:

  • text
  • email
  • number
  • multiline
  • select
Fetch data from
GET https://vb-react-exam.netlify.app/api/form
Sample Response
{
    "success": true,
    "message": "Fetch success!",
    "data": [
        {
            "fieldName": "firstName",
            "type": "text",
            "value": "Winfield"
        },
        ...
        {
            "fieldName": "gender",
            "type": "select",
            "value": "female",
            "options": [
                "male",
                "female",
                "other"
            ]
        }
    ]
}

2. Send form data

User should be able to send the data after editing the form and print the response on the page.

API endpoint
POST https://vb-react-exam.netlify.app/api/form
Request Body
  • firstName - string
  • lastName - string
  • emailAddress - string
  • [randomString] - string (fieldName randomly generated by API)
  • gender string (optional) - male | female | other
  • age - number (optional)
  • testimonial - string (optional)
Sample Response
{
    "success": true,
    "message": "Submission success!",
    "data": {
        "firstName": "asd",
        "lastName": "1",
        "emailAddress": "jere@asd.com",
        "gender": "other",
        "age": "26"
    }
}

Sample

See sample app for reference: https://vb-react-exam.netlify.app

Final Notes

  • You can run the application on docker/docker-compose but is optional
  • You can use any state management solution (redux or zustand is a plus)
  • Styling is not necessary but use of material-ui is encouraged
  • User experience is important - Disable buttons/fields when loading or show loading indicators
  • Treat the project as for production
    • important variables/information should not be visible on code
    • use git to track changes
  • You can use any third party library as you see fit
